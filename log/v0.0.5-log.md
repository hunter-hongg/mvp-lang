# 🎉 Miva v0.0.5 Released

> **From pleasant to practical**

We’re pleased to announce the release of **Miva v0.0.5**!  
This version focuses on **solidifying expressive power, expanding standard capabilities, and paving the way for a mature toolchain**—taking Miva from “fun to experiment with” to “ready for real use.”

---

## 🔧 Core Updates

### 1. **`elif` Syntax Sugar**

Multi-branch conditionals are now cleaner and more readable:

```miva
if x < 0 {
    log("negative")
} elif x == 0 {
    log("zero")
} else {
    log("positive")
}
```

Semantically identical to `else { if ... }`, with zero runtime cost—purely for improved clarity.

---

### 2. **Native Pointer Types: `box<T>` and `ptr<T>`**

- **`box<T>`**: A uniquely owned heap pointer. Move-only, non-copyable.  
  → Compiles to C++ `std::unique_ptr<T>`.
- **`ptr<T>`**: An unsafe raw pointer. May only be dereferenced inside `unsafe` blocks.  
  → Compiles to C++ `T*`.

These enable FFI, custom data structures, and systems-level programming—while keeping memory safety boundaries explicit.

---

### 3. **Module Aliasing with `import as`**

Resolving naming conflicts is now straightforward:

```miva
import "std/json" as sjson
import "mylib/json" as tjson

sjson.parse(...) // unambiguous origin
```

Wildcard imports are also supported (use sparingly):

```miva
import "std/macro" as .  // imports all symbols into the current scope
```

---

### 4. **Subdirectory Build Support**

Project layout is now more flexible:

```bash
miva init myapp --type=bin
cd src/
miva build  # ✅ Now valid!
```

The compiler automatically walks up the directory tree to locate `miva.toml`, so you no longer need to return to the project root to build.

---

### 5. **`--release` Build Mode**

Performance-critical builds now have a dedicated path:

```bash
miva build            # debug: includes debug symbols, no optimizations
miva build --release  # release: -O2 -DNDEBUG, debug info stripped
```

Also sets the compile-time constant `MIVA_DEBUG = "false"` for conditional compilation.

---

### 6. **Built-in Macro System (v1)**

The first set of read-only, hygienic macros is now available:

```miva
printlns!("hello", "world")        // prints each argument
assert!(x > 0)
assert_eq!(a, b)
let content = include_str!("data.txt")  // embeds file at compile time
```

- User-defined macros are not yet supported (planned for v0.1.0).
- Macros are fully hygienic—no variable capture, no scope pollution.

---

## 🚫 Not Included in This Release

- User-defined macros
- Static library builds (`--static`)
- LSP or IDE plugins
- Concurrency models (threads / async)

> We believe: **Every step should be deliberate; every line, verifiable.**
